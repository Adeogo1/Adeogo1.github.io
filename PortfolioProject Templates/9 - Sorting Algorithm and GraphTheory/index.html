Intro

Graphs allow us to visualize the problem using nodes to represent an article of clothing and edges to represent relationships between them. Many real world situations can be modelled as a graph with directed edges where some events must occur before others e.g. school class prerequisites, program dependencies, event scheduling, Assembly instructions etc

There are different types of graphs.

Undirected graph - An undirected graph is a type of graph in which the edges have no direction, meaning that they connect two vertices in both directions. This is in contrast to a directed graph, in which the edges have a direction and connect one vertex to another in a specific direction. In an undirected graph, if vertex A is connected to vertex B, then vertex B is also connected to vertex A. This means that there is no distinction between the "starting" and "ending" vertex of an edge, and the edges are often represented as simple lines connecting the vertices.

Directed Graphs - A directed graph, also known as a digraph, is a graph data structure in which edges have a direction and are represented by arrows. Each vertex (or node) in a directed graph has a set of incoming edges and a set of outgoing edges, indicating the direction of the connections between the vertices. Directed graphs are commonly used to represent relationships between objects or entities where the direction of the relationship is important, such as in a social network or a transportation network. They can be used to model many real-world systems, such as computer networks, transportation systems, and biological systems.

Weighted Graphs - A weighted graph is a graph in which each edge has a numerical value, called a weight. These weights can represent various things such as distance, cost, or even a probability. Weighted graphs can be used to model real-world problems such as finding the shortest path between two points, or the most cost-effective path in a network. These graphs are useful in solving problems involving optimization and decision-making. They can be either directed or undirected, depending on the problem and context.

Trees - a tree is a data structure that represents a hierarchical relationship between elements, called nodes. Each node in a tree can have one or more child nodes, and each child node can have one or more children of its own, and so on. The top node in a tree is called the root, and nodes that do not have children are called leaf nodes. Trees are used in many different applications, including file systems, databases, and algorithms for searching and sorting data.

A Rooted Tree - A rooted tree is a specific type of tree data structure in computer programming where one node, called the root, serves as the topmost node and parent of all other nodes in the tree. All other nodes in the tree are connected to the root, and each node can have one or more child nodes. The tree structure is hierarchical, meaning that each node can have a parent node and child nodes, with the exception of the root node which has no parent. Rooted trees are commonly used to represent hierarchical relationships between data

Directed Acyclic Graphs (DAGs) - A Directed Acyclic Graph (DAG) is a type of directed graph that contains no directed cycles, meaning that there is no way to start at any vertex in the graph and follow a sequence of directed edges that eventually loops back to the starting vertex. In other words, it is a directed graph without any loops. DAGs are often used to model relationships between events or tasks, where the edges represent causal relationships and the absence of cycles ensures that there are no paradoxes or circular dependencies

Work

Sorting Algorithm - SelectionSort, InsertionSort, QuickSort, MergeSort, HeapSort, RadixSort, BubbleSort, BitonicSort

Graph Algorithm - Adjacency Matrix, Adjacency List, Topological sort, Bellman-Ford, Algorithm to find Bridges and Articulation Points, Tarjan’s Algorithm, Prim’s Minimum Spanning Tree, LinkedList

A adjacency Matrix is a very simple way to represent a graph. The idea is that the cell m[i][j] represents the edge weight of going from node i to node j

An adjacency list - a way to represent a graph as a map from nodes to lists of edges

Topological sort - a topological ordering is an ordering of nodes in a directed graph where for each directed edge from node A to node B, node A appears before node B in the ordering. The topological sort algorithm can find a topological ordering

Bellman-Ford - A single source shortest path algorithm. this means it can find the shortest path from one node to any other node. It is not ideal for most SSSP problems because it has high time complexity
